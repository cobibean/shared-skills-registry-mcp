# Demo Script

The first demo should prove one clean path:

> Publish a skill → see it in the UI → retrieve it over MCP → install it locally.

## Scene 1 — The problem

Agents are accumulating useful skills, but those skills live in scattered folders. Reusing one across multiple agents or devices usually means copy/paste, stale versions, and no visibility.

## Scene 2 — Add/import a skill

Add one generic public-safe demo skill into the registry. The skill should include a `SKILL.md` and at least one supporting file so the bundle model is visible.

## Scene 3 — Browse it in the dashboard

Show the dashboard listing skills, filtering/searching, and opening a detail page with metadata, files, validation state, and install/readiness information.

## Scene 4 — Retrieve over MCP

Show an MCP-compatible agent listing, searching, describing, and retrieving the skill bundle.

## Scene 5 — Install locally

Install the skill into a configured local skill directory through a safe adapter. Show path validation, checksum verification, and an audit entry.

## Scene 6 — Wrap the story

"This is a package manager/control panel for reusable AI-agent skills: self-hosted, visible, MCP-native, and designed not to expose your private fleet."
